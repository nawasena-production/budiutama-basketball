09 Firestore Schema & Document ID Convention

Struktur Collection Lengkap

users/ coach_budi, manager_andi, player_7_ar teams/ putra_2526, putri_2526 players/ 7_ar_putra2526 // #7 Ahmad Rizki 4_dp_putra2526 // #4 Dimas Pratama 3_rn_putri2526 // #3 Rina Nuraini events/ porseni_kota_2526 friendly_sman1_2526 matches/ porseni_kota_2526_vs_sman1_20260315 { current_state, home_score, opponent_score, quarter_duration_minutes, num_periods, timer_config_locked, ... } ├── timer_state // dokumen tunggal │ { is_running, seconds_at_start, │ started_at (ServerTimestamp), quarter } ├── events/ │ q1_001, q1_002, ..., q3_047 ├── player_stats/ │ 7_ar { points, fg2_made, shot_zones, ... } │ 4_dp { points, rebounds, ... } └── lineups/ 7_ar { is_on_court, total_seconds_played, ... } physical_test_sessions/ beep_putra2526_20260120 └── results/ 7_ar, 4_dp { beep_level, beep_shuttle } injury_reports/ 7_ar_putra2526_20260205 training_sessions/ putra2526_fisik_20260110 audit_logs/ // auto-generated ID oleh Cloud Functions

Format ID Convention

CollectionFormat ID
users{role}_{nama}
teams{gender}_{tahun}
players{jersey}_{inisial}_{team}
events{tipe}_{nama}_{tahun}
matches{eventId}_vs_{lawan}_{tgl}
match/eventsq{quarter}_{urutan3digit}
match/player_stats{jersey}_{inisial}
physical_test_sessions{tipe}_{team}_{tgl}
injury_reports{playerId}_{tgl}
training_sessions{team}_{tipe}_{tgl}
Aturan penamaan:
• Semua lowercase
• Gunakan underscore (_), bukan spasi atau slash
• Tahun ajaran 2025/2026 → 2526
• Tanggal → YYYYMMDD (sortable)
• Contoh: 15 Maret 2026 → 20260315

13 Interaction Design Rules

Prinsip UX Match Mode (Flutter)

AturanDetail
Max 2 tap untuk aksi kritisPilih pemain → tap aksi. Selesai.
Touch target min 44×44dpSemua tombol aksi di Match Mode
Visual feedback < 100msRiverpod state update segera; Firestore write async
Konfirmasi wajibTransisi state, undo, substitusi — pakai showDialog()
Koneksi status selalu terlihatIndikator Firestore sync di header — tidak boleh tersembunyi
Tidak ada scroll di Match ModeSemua elemen fit dalam satu layar landscape tablet
Shot clock tidak adaSistem untuk penggunaan internal — bukan kompetisi resmi

Prinsip UX Dashboard (Flutter)

AturanDetail
Bottom sheet untuk formGunakan showModalBottomSheet() bukan full-screen form
Snackbar untuk feedbackSukses/gagal via ScaffoldMessenger.showSnackBar()
Destructive action: 2 langkahHapus/nonaktifkan perlu showDialog() konfirmasi
Loading stateCircularProgressIndicator atau Shimmer skeleton
Empty state informatifGambar + teks + CTA button, bukan halaman kosong
Pull-to-refreshRefreshIndicator untuk halaman list

Firebase Offline Behavior

KondisiBehavior Flutter + Firestore
Koneksi baikWrite langsung ke Firestore server; listener realtime
Koneksi terputusWrite ke Firestore local cache; UI tetap update
Indikator offlineConnectionStatus widget di header Match Mode: 🔴 Offline
Koneksi kembaliFirestore otomatis sync cache ke server; listener resume
Conflict resolutionLast-write-wins (Firestore default)
